Storage Account

  1. An Azure Storage Account is a cloud-based service provided by Microsoft Azure that offers highly available, secure and scalable storage for various types of data

  2. It has multiple storage types, including blobs (unstructured data like documents and media), files, queues (message storage) and tables (structured NoSQL data)

  3. Azure Storage Accounts are designed to handle a wide range of data storage needs for applications running in the cloud, on-premises, or hybrid environments

When to use Storage Account​

Considering selecting an Azure Storage Account for

Deciding FactorsReason
Storing Large Amounts of DataIdeal for handling large volumes of unstructured data, such as media files, logs, backups and datasets.
High Availability and Durability NeedsDesigned for 99.9% availability and up to 16 nines durability depending on the redundancy option selected.
Cost EfficiencyOffers multiple storage tiers (hot, cool and archive) to optimize costs based on data access frequency.
Flexible Data Access PatternsSupports a wide range of use cases, from high-throughput analytics to low-latency web applications.
Integration with Azure ServicesSeamlessly integrates with other Azure services like Azure Virtual Machines, Azure Functions, Azure App Service and Azure Synapse Analytics.
Data Compliance and SecurityEnsures compliance with various regulations and standards (e.g., GDPR, HIPAA) through features like encryption and data protection.

2. Performance Tiers​

FactorsReason
Hot TierOptimized for storing data that is accessed or modified frequently. It has the highest storage costs but the lowest access costs.
Cool TierOptimized for storing data that is infrequently accessed or modified. Data should be stored for a minimum of 30 days. It has lower storage costs and higher access costs compared to the hot tier.
Cold TierOptimized for storing data that is rarely accessed or modified but requires fast retrieval. Data should be stored for a minimum of 90 days. It has lower storage costs and higher access costs compared to the cool tier.
Archive TierOptimized for storing data that is rarely accessed and has flexible latency requirements, on the order of hours. Data should be stored for a minimum of 180 days.

3. Redundancy Options​

FactorsReason
Locally Redundant Storage (LRS)Replicates your storage account three times within a single data center in the primary region.
Zone-Redundant Storage (ZRS)Replicates your storage account synchronously across three Azure availability zones in the primary region.
Geo-Redundant Storage (GRS)Copies data synchronously three times within a single physical location in the primary region using LRS, then copies data asynchronously to a single physical location in the secondary region. Within the secondary region, your data is copied synchronously three times using LRS.
Geo-Zone-Redundant Storage (GZRS)Copies your data synchronously across three Azure availability zones in the primary region using ZRS. It then copies your data asynchronously to a single physical location in the secondary region. Within the secondary region, your data is copied synchronously three times using LRS.
Read-Access Geo-Redundant Storage (RA-GRS)Same as GRS, with read access to the secondary region. Learn more.

4. Access Controls​

FactorsReason
Network RestrictionsUse firewalls, virtual network (VNet) service endpoints, or private endpoints to restrict access.
Identity-Based AccessLeverage Microsoft Entra ID for secure, managed identity access.

Best Practices​

1. Network Security​

FactorsReason
Network RestrictionsUse firewall rules, virtual network (VNet) service endpoints to restrict access to the storage account.
Private EndpointsConfigure private endpoints to ensure that traffic between your VNet and Azure Storage travels over the Azure backbone network.

2. Data Encryption​

FactorsReason
At RestAzure Storage automatically encrypts data at rest using Storage Service Encryption (SSE) with 256-bit AES encryption.
In TransitUse HTTPS to ensure secure data transfer over the network.
Customer-Managed KeysUse Azure Key Vault to manage encryption keys.

3. Access Control​

FactorsReason
Shared Access Signatures (SAS)Use SAS tokens to grant limited access to resources without exposing your storage account keys. Note: Always use RBAC-based access to the storage account over SAS.
Role-Based Access Control (RBAC)Implement RBAC to control access to resources within the storage account.

4. Data Protection​

FactorsReason
Soft DeleteEnable soft delete for blobs and files to recover deleted data.
VersioningEnable versioning to keep track of changes to blobs and recover previous versions if needed.

5. Monitoring and Alerts​

FactorsReason
Azure MonitorUse Azure Monitor to track metrics and set up alerts for storage account performance and health.
Activity LogsMonitor activity logs to track operations and diagnose issues.

6. Configuration​

FactorsReason
Replication StrategyChoose the appropriate redundancy option based on your disaster recovery and high availability needs.
Access TiersSelect the access tier that aligns with your access patterns and cost considerations.
Geo-ReplicationDecide if geo-replication is necessary based on your data residency and compliance requirements.

7. Cost Management​

FactorsReason
Storage CostsMonitor and manage costs based on the storage tier and redundancy option selected.
Lifecycle ManagementImplement lifecycle management policies to move data to lower-cost storage tiers or delete data automatically.
